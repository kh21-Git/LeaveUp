계정 기능 설정 방법
  1. supabase.com 프로젝트의 Authentication → Providers → Email을 켜고, Confirm email(이메일 인증)은 끕니다.
  2. SQL Editor에서 아래를 실행해 계정별 데이터 테이블·보안 정책을 만듭니다. 
    create table if not exists public.leave_data (
  user_id uuid primary key references auth.users(id) on delete cascade,
  data jsonb,
  updated_at timestamptz default now()
);
alter table public.leave_data enable row level security;
create policy "own_select" on public.leave_data
  for select using (auth.uid() = user_id);
create policy "own_insert" on public.leave_data
  for insert with check (auth.uid() = user_id);
create policy "own_update" on public.leave_data
  for update using (auth.uid() = user_id) with check (auth.uid() = user_id);
create policy "own_delete" on public.leave_data
  for delete using (auth.uid() = user_id);
  3. Project Settings → API Keys 의 Project URL과 Publishable key(sb_publishable_… / 예전 프로젝트는 anon key)를 이 파일 상단 SB_CFG의 url·key에 붙여넣습니다.
  4. 파일을 올리면 접속 시 로그인 화면이 뜨고, 이메일·비밀번호로 가입/로그인하면 계정별로 데이터가 저장됩니다.
  5. 비밀번호 재설정을 쓰려면 Authentication → URL Configuration에서 Site URL을 이 앱 주소(예: https://kh21-git.github.io/LeaveUp/)로 설정하고, Redirect URLs에도 같은 주소를 추가합니다. (기본 제공 메일은 발송량 제한이 있어, 사용자가 많으면 SMTP 연결을 권장합니다.)
