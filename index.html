클라우드 동기화 (외부·다른 기기 접속)

Supabase 프로젝트를 연결하면 데이터가 클라우드에 저장되어, 외부나 다른 기기에서도 같은 URL·키·동기화 ID로 접속해 이어서 사용할 수 있습니다. 비워두면 이 브라우저에만 저장됩니다.

연결되지 않음
설정 방법 보기
  1. supabase.com 에서 무료 프로젝트를 생성합니다.
  2. SQL Editor에서 아래를 실행해 테이블을 만듭니다. 
    create table if not exists public.leave_sync (
  id text primary key,
  data jsonb,
  updated_at timestamptz default now()
);
-- 새 프로젝트는 테이블을 API에 자동 노출하지 않으므로 권한을 직접 부여
grant select, insert, update, delete on table public.leave_sync to anon;
-- 개인용 간이 사용: RLS 비활성화 (동기화 ID를 키처럼 사용)
alter table public.leave_sync disable row level security;
  3. Project Settings → API Keys 에서 Project URL과 Publishable key(sb_publishable_…, 예전 프로젝트는 anon key)를 복사해 위 칸에 붙여넣습니다.
  4. 동기화 ID에 본인만 아는 값을 입력하고 [연결 저장]을 누릅니다.
  5. 다른 기기에서도 같은 URL·키·동기화 ID를 입력하면 데이터가 이어집니다.

※ 이 방식은 anon 키가 화면에 포함되므로 강력한 보안이 아닙니다. 동기화 ID는 추측하기 어려운 값으로 설정하세요. 민감한 조직 데이터에는 서버 측 인증(RLS 등)이 필요합니다.
